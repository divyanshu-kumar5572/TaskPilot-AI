# 🚀 TaskPilot AI - Your Day, AI Optimized

TaskPilot AI is an elegant, full-stack, AI-powered scheduling and task management assistant. It automatically transforms a simple list of tasks, habits, and preferences into a fully structured daily schedule, and seamlessly synchronizes those events to your **Google Calendar** using real-time Firebase Authentication and OAuth.

---

## ✨ Features

- **🧠 Smart AI Daily Planner**: Generate optimal, color-coded daily agendas from basic lists, taking into account priority, break times, and productivity habits.
- **📅 Google Calendar Synchronization**: Instantly export your generated schedules into your real Google Calendar as real events.
- **🔒 Firebase-Powered Authentication**: Secure login using Google Sign-In, enabling cross-device schedule persistence and safe API handshakes.
- **💫 Modern, High-Contrast Glassmorphic UI**: Beautiful, eye-safe slate-colored workspace built with React, Vite, Tailwind CSS, and Framer Motion transitions.

---

## 🛠️ Prerequisites

Before running the project locally, make sure you have:

1. **Node.js** (v18.x or higher) installed on your system.
2. A **Firebase Project** (on the [Firebase Console](https://console.firebase.google.com/)).
3. A **Google Cloud Console Project** (automatically created with Firebase or configured at [Google Cloud Console](https://console.cloud.google.com/)).

## ⚡ Quick Start: Local Installation

Follow these steps to run TaskPilot AI on your local computer:

### 1. Clone the Repository
```bash
git clone <your-github-repo-url>
cd <your-project-directory>
```

### 2. Install Dependencies
```bash
npm install
```

### 3. Configure Environment Variables
Create a file named `.env` in the root of the project and fill in your keys (use `.env.example` as a template):

```env
# Google Gemini API key for smart AI scheduling
GEMINI_API_KEY="your-gemini-api-key-here"

# Your local app hosting address
APP_URL="http://localhost:3000"
```

> 💡 **How to get a Gemini API Key:** You can generate a free Gemini API key in seconds from the [Google AI Studio website](https://aistudio.google.com/).

## 🔑 Firebase & Google OAuth Integration Setup

To get Google Sign-In and Google Calendar Export working perfectly on your custom deployment, you must link it to your own Firebase and Google Cloud accounts:

### Part A: Set up your Firebase App
1. Go to the [Firebase Console](https://console.firebase.google.com/).
2. Create a new project (e.g., "TaskPilot AI").
3. Go to **Project Settings** (gear icon) and under **Your Apps**, add a **Web App** to generate your Firebase configuration.
4. Replace the configuration in `/src/firebase.ts` with your new credentials:
   ```typescript
   const firebaseConfig = {
     apiKey: "YOUR_API_KEY",
     authDomain: "YOUR_AUTH_DOMAIN",
     projectId: "YOUR_PROJECT_ID",
     storageBucket: "YOUR_STORAGE_BUCKET",
     messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
     appId: "YOUR_APP_ID"
   };
   ```
5. In the Firebase left-side panel, go to **Build** -> **Authentication** and click **Get Started**.
6. Enable **Google** as a Sign-In provider under the **Sign-in method** tab.

### Part B: Enable Google Calendar API & Config OAuth
1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
2. Select the Google Cloud project associated with your Firebase App (they share the same Project ID).
3. In the search bar at the top, search for **Google Calendar API** and click **Enable**.
4. Go to **APIs & Services** -> **OAuth Consent Screen**:
   - Set the User Type to **External**.
   - Fill out the App Name, User support email, and developer contact details.
   - **Crucial step (Scopes)**: Click **Add or Remove Scopes** and add the following scope to enable calendar writing:
     * `https://www.googleapis.com/auth/calendar.events` (or simply search "Calendar API" and select the permission to edit/manage events).
   - In the **Test Users** screen, add your personal Gmail address (the one you will sign in with). This is required until you request public verification from Google.
5. Go to **APIs & Services** -> **Credentials**:
   - Look for the **OAuth 2.0 Client IDs** automatically generated by Firebase (often named `Web client (auto created by Google Service)`).
   - Edit this client ID and add your authorized domains:
     * **Authorized JavaScript Origins**: `http://localhost:3000` (and your production URL if deployed).
     * **Authorized Redirect URIs**: `https://<your-firebase-project-id>.firebaseapp.com/__/auth/handler` (this enables Firebase Authentication redirects).

## 💻 Running the Server

TaskPilot AI is designed as a full-stack application (Express + Vite) to keep all API keys hidden safely on the server side:

### Run in Development Mode
This starts the local development environment on port `3000` with hot-reloading:
```bash
npm run dev
```
Open your browser to: **`http://localhost:3000`**

### Build and Run for Production
To bundle the frontend single-page app and compile the Express server into a standalone optimized build:
```bash
# Build the code
npm run build

# Start the compiled production server
npm run start
```

## 🏗️ Project Architecture

- `/src/components/`: Modular React components (such as `AISchedulePanel`, `Header`, `Calendar`, and custom Modals).
- `/src/calendarService.ts`: Core module handling the Google Calendar API requests (fetching current schedules, creating calendars, and posting batch events).
- `/src/firebase.ts`: Direct client integration for Firebase Authentication.
- `/server.ts`: Full Express backend serving compiled assets and hosting the Gemini API proxy to keep secrets secure.
